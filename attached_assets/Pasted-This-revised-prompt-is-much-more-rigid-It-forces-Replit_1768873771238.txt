This revised prompt is much more rigid. It forces Replit to follow your specific flow (Email -> Title -> Calendar -> Share) and ensures the UI matches your "Full Day/Evening" requirement.

I've also added a specific "Matrix" view requirement, which is the "Doodle" standard for seeing who voted for what at a glance.

The Prescriptive Replit Prompt
Act as an expert Full-Stack Engineer. Build a Group Scheduling Web App (Doodle Clone) using Flask, SQLite, and Tailwind CSS. The app must be highly portable and self-contained.

1. Data Model & Logic
Database (SQLite): Create tables for Polls, Dates (specific dates attached to a poll), and Votes (linked by user_email, date_id, and status).

Vote Status: Must support three states: yes, no, and maybe.

Security: Use session-based authentication. A user "logs in" by entering their email; this email is then used to track all their interactions.

2. The Prescriptive User Flow
Step 1: Admin Initialization

Home page is a simple "Create a Poll" form.

Fields: Admin Email and Poll Name.

Clicking "Next" saves these to the session and moves to the calendar.

Step 2: Date Selection (The Admin View)

Provide a full-screen Calendar interface (use FullCalendar.js).

Users click/tap dates to toggle them "On" (selected for the poll).

No time slots—selection is for the full day only.

Users must be able to browse months into the future.

CTA: A "Create and Share" button that finalizes the poll.

Step 3: Sharing Screen

Once created, show a success page.

Field: A Read-only URL to the poll (e.g., /poll/<uuid>) with a "Copy to Clipboard" button.

Field: An optional "Invite via Email" text area where emails can be listed (just for storage/reference).

Step 4: The Voting View (The Participant View)

If a visitor isn't logged in via email, show an "Enter your email to vote" overlay.

The Matrix Grid: Display a table where:

Rows: Participant Email addresses.

Columns: The dates selected by the Admin.

Cells: Show the status (Yes/No/Maybe) for each person.

Voting Action: Below the grid, the current user has a row of toggle buttons for each date to set their own status.

Step 5: Results & Dashboard

On the voting page, highlight the column (date) that has the most "Yes" votes. If there is a tie, highlight all tied dates.

Provide a "My Polls" dashboard at /dashboard that lists all polls created by or voted on by the logged-in email, sorted by most recent.

3. Technical Constraints & UI
Architecture: Ensure all backend logic is in main.py and frontend in a templates/ folder.

UI Style: Minimalist, clean, and mobile-responsive using Tailwind CSS.

Visuals: Use Green for "Yes", Red for "No", and Yellow/Amber for "Maybe".

Persistence: Ensure the SQLite database file is created automatically on first run if it doesn't exist.
